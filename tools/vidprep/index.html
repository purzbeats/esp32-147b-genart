How the preview maps to the device

The big preview is exactly 172×320 pixels scaled up with nearest-neighbour (no smoothing), so you see the true pixels the panel will show — not interpolated soup. The downscale filter picks how the crop is squeezed down to 172×320, and it's matched into the ffmpeg scale flags. Pipeline order (preview & ffmpeg both): rotate → crop → scale 172×320 → fps.

  • The crop box is locked to the panel's 172:320 (tall) aspect. For a landscape video, rotate 90° so it fills the screen instead of becoming a thin centre strip.
  • Audio is dropped (-an) — no speaker on the board.
  • Trim uses -ss + -t (start + duration) for unambiguous cuts.
